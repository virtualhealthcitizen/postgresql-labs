# Exercises: PostgreSQL `IN` Operator

## Setting Up a Temporary Test Database

To practice using the `IN` operator, we will create a temporary test database and a table called `test_rental`.

### 1. Creating a Temporary Test Database

```sql
CREATE DATABASE testdb;
```

### 2. Connecting to the Temporary Test Database

```bash
\c testdb
```

### 3. Creating the `test_rental` Table and Inserting Data

```sql
CREATE TABLE test_rental (
    rental_id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL,
    return_date DATE
);

INSERT INTO test_rental (customer_id, return_date)
VALUES
(1, '2023-01-01'),
(2, '2023-01-02'),
(3, '2023-01-03'),
(1, '2023-01-04'),
(2, '2023-01-05'),
(3, '2023-01-06');
```

## Exercises

### Exercise 1: Using the `IN` Operator

**Objective**: Retrieve rental information for customer IDs **1** and **3**.

**Task**: Use the `IN` operator in the `WHERE` clause.

```sql
SELECT customer_id, rental_id, return_date
FROM test_rental
WHERE customer_id IN (1, 3)
ORDER BY return_date DESC;
```

### Exercise 2: Using the `NOT IN` Operator

**Objective**: Retrieve rental information for customers other than IDs **1** and **3**.

**Task**: Use the `NOT IN` operator in the `WHERE` clause.

```sql
SELECT customer_id, rental_id, return_date
FROM test_rental
WHERE customer_id NOT IN (1, 3)
ORDER BY return_date DESC;
```

### Exercise 3: Using the `IN` Operator with a Subquery

**Objective**: Retrieve rental information for customers who have rentals on `2023-01-02`.

**Task**: Use the `IN` operator with a subquery in the `WHERE` clause.

```sql
SELECT customer_id, rental_id, return_date
FROM test_rental
WHERE customer_id IN (
    SELECT customer_id
    FROM test_rental
    WHERE return_date = '2023-01-02'
)
ORDER BY customer_id;
```

### Exercise 4: Using the `IN` Operator with Multiple Values

**Objective**: Retrieve rental information for customer IDs **1**, **2**, and **3**.

**Task**: Use the `IN` operator with multiple values in the `WHERE` clause.

```sql
SELECT customer_id, rental_id, return_date
FROM test_rental
WHERE customer_id IN (1, 2, 3)
ORDER BY return_date DESC;
```

## Cleaning Up

After completing the exercises, you may want to clean up by dropping the test database:

```sql
DROP DATABASE testdb;
```

## Summary

- The `IN` operator checks if a value matches any value in a list of values.
- The `NOT IN` operator selects rows whose values do not match the values in the list.
- The `IN` operator can be used with subqueries to filter rows based on the results of another query.
- Using the `IN` operator is often more readable and efficient than using multiple `OR` conditions.
